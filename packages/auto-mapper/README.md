# auto-mapper

Browser-injectable DOM perception module (Phase B, Bill product path).

Given a live DOM, produces:
1. A **PageNode** — url, title, archetype guess, landmarks, headings, affordances (links/buttons/inputs with role + accessible name + stable selector + co-selector anchors per VISION.md stability hierarchy)
2. **Candidate walkthrough steps** for the page's primary affordances
3. Both output formats: **Yeshie `site.model.json` fragment** and **SomaGuideConfig**

No LLM calls in v1 — heuristic perception only.

---

## Files

| File | Purpose |
|---|---|
| `auto-mapper.js` | Perception module. `perceive(doc, win)` → PageNode, `generateSteps(pageNode, personaName)` → steps |
| `converter.js` | Format converter. PageNode ↔ Yeshie fragment ↔ SomaGuideConfig |

---

## Usage

### In a browser / extension

```html
<script src="vendor/auto-mapper.js"></script>
<script src="vendor/converter.js"></script>
<script>
  const pageNode = AutoMapper.perceive(document, window);
  const config   = AutoMapperConverter.pageNodeToSomaConfig(pageNode, {
    name: 'Ariadne', avatar: '🧵', greeting: 'Hi!',
  });
  window.SomaGuideConfig = Object.assign(config, {
    voiceAgentId: 'agent_...',
    ttsProxyUrl:  'https://...',
    inferenceUrl: 'https://...',
    askFirst:     true,
  });
</script>
```

### In Node (tests)

```js
const AutoMapper = require('./auto-mapper.js');
const Converter  = require('./converter.js');

const pageNode = AutoMapper.perceive(dom.window.document, dom.window);
const steps    = AutoMapper.generateSteps(pageNode, 'Ariadne');
const fragment = Converter.pageNodeToYeshieFragment(pageNode, steps);
const config   = Converter.yeshieFragmentToSomaConfig(fragment);
```

## Format Mapping: Yeshie site.model.json ↔ SomaGuideConfig

| Yeshie fragment field | SomaGuideConfig field | Notes |
|---|---|---|
| `_meta.site` | `siteMap[0].id` | hostname |
| `_meta.archetypeGuess` | `siteMap[0].archetype` | heuristic |
| `_meta.framework` | (not in SomaGuideConfig) | informational only |
| `navigation[]` | `siteMap[0].links[]` | `{text, href}` → `{label, url}` |
| `stateGraph.nodes` | `siteMap[]` | one entry per page node |
| `walkthroughAnnotations[]` | `walkthroughs[0].steps[]` | 1-to-1 step mapping |
| `abstractTargets[id]` | (not in SomaGuideConfig) | Yeshie-only; resolution cache |
| `pages[key].buttons` | `walkthroughs[0].steps[n].target` | via affordances → steps |
| `_meta.autoGenerated: true` | — | marks auto-generated fragments |

### PageNode → Yeshie fragment mapping

| PageNode field | Fragment field |
|---|---|
| `url` | `_meta.site` (hostname), `stateGraph.nodes[key]` signals |
| `title` | `_meta.name` prefix, `stateGraph.nodes[key].description` |
| `archetype` | `_meta.archetypeGuess` |
| `framework` | `_meta.framework` |
| `mappedAt` | `_meta.lastExplored` |
| `headings` | `stateGraph.nodes[key].knownElements`, `pages[key].headings` |
| `landmarks` | `pages[key].landmarks` |
| `affordances[role=link]` | `navigation[]` |
| `affordances[role=button]` | `pages[key].buttons[]` |
| `affordances[role=textbox]` | `pages[key].inputs[]` |
| `affordances` (all) | `abstractTargets[slug]` with selector + anchors |
| `primaryAffordances` | `capabilities[key].actions[]` |
| `formCount` | `pages[key].formCount` |
| `generateSteps()` output | `walkthroughAnnotations[]` |

## Archetype classification

Heuristic signals (no LLM):

| Archetype | Signals |
|---|---|
| `admin-dashboard` | `.v-application`, `/admin\|dashboard/` in title/url, sidebar nav |
| `auth-page` | `/sign.?in\|log.?in\|password\|register/` in title/url |
| `content-site` | `/blog\|article\|post\|news/` in title/url |
| `marketing-site` | `/cta\|hero\|pricing\|features\|landing/` in title/url |
| `form-page` | 1+ forms and ≤3 headings |
| `generic` | fallback |

**LLM_HOOK**: Search for `LLM_HOOK` in `auto-mapper.js` for the two marked extension points where an LLM pass could enrich classification and step labeling.

## Selector stability hierarchy

Follows VISION.md §"Selector Stability Hierarchy" (most → least durable):

1. `aria-label` → `[aria-label="..."]`
2. `placeholder` → `tag[placeholder="..."]`
3. `name` attribute → `tag[name="..."]`
4. `data-testid` → `[data-testid="..."]`
5. Non-generated `id` → `#id`
6. Anchor `href` → `a[href="..."]`
7. Stable developer-named class → `.classname`
8. `null` (no stable selector found)

Co-selector anchors (`ariaLabel`, `placeholder`, `name`, `dataTestId`, `id`, `text`) are stored alongside the primary selector for auto-heal recovery when the primary breaks (e.g. on re-bundle).

## Manual test recipe

1. Open `chrome://extensions` → Enable Developer mode → Load unpacked → select `packages/guide-extension/`
2. Navigate to [wolfeducationalconsulting.com](https://wolfeducationalconsulting.com) (or any unconfigured page)
3. Click the 🧵 toolbar button
4. Confirm Ariadne's widget appears with a generated tour
5. Open DevTools console and inspect:
   - `window._somaAriadnePageNode` — full PageNode from AutoMapper
   - `window._somaAriadneMap` — simplified map (backwards compat + devtools)
   - `window.SomaGuideConfig` — generated SomaGuideConfig
6. Click "Take a tour" — verify steps navigate through the page's primary affordances

**Site-aware hand-off test**: On a page that already sets `window.SomaGuideConfig` (e.g. bill-talk.netlify.app), clicking 🧵 should use the native config and NOT overwrite it with the auto-generated one.

## Open questions

1. **LLM refinement pass**: should the archetype classifier and step narration generator call an LLM endpoint? The `LLM_HOOK` points are clearly marked; v1 is heuristic-only to avoid latency on every toolbar click.

2. **Yeshie fragment persistence**: currently the fragment is only held in `window._somaAriadnePageNode`. Should the extension POST it to a mapper endpoint for accumulation into a shared site model? Deferred to Phase C (cold-site falsifier).

3. **Multi-page exploration**: the auto-mapper only scans the current page. Phase C will add autonomous cross-page exploration to build a complete L3 site model.

4. **Form interaction steps**: inputs and forms are detected as affordances but steps for them are minimal. A richer pass could inspect field labels and types to generate fill-in steps (e.g. "Type your name here").

5. **CSS.escape compatibility**: `auto-mapper.js` calls `CSS.escape()` for stable id selectors. In very old browsers this may fail; a polyfill is included in `perceive.js` for jsdom tests but not in the vendor build.
